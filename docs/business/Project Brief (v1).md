# ProPhoto — Project Brief (v1)

## Core Idea

ProPhoto is a workflow-first photography platform that eliminates post-shoot decision fatigue by turning scattered tools into a single intelligent system.

It connects:

- Planning (Calendar / Session Context)
- Capture (Ingest + Metadata)
- Delivery (Gallery + Client Interaction)

…into one continuous flow.

---

## The Defining Moment

> A photographer uploads hundreds of raw files…

…and instead of organizing them manually, ProPhoto responds:

“Looks like the Alma Mater Footwear Lifestyle shoot went well.  
You finished 15 minutes early.  
Are we culling these now or moving to delivery?”

This is the product.

- It understands the shoot
- It connects images to real-world context
- It suggests the next action

---

## The 3 Pillars

### 1. Context Engine (Calendar + Session Awareness)

- Uses calendar (when available) to define:
  - who (client, subjects)
  - where (location)
  - what (session type)
- Pre-stages galleries before images exist
- Eliminates post-shoot data entry

If no calendar exists:
- system falls back to metadata + user input

---

### 2. Pro Ingest (Control Layer)

- High-performance ingest pipeline
- Tagging, grouping, and culling during upload
- Metadata-driven filtering (EXIF + user signals)
- Enables work to begin immediately

This replaces the traditional “import → wait → organize” workflow.

---

### 3. Delivery Layer (Gallery System)

- Fast, clean, client-ready galleries
- Subject + client interactions:
  - selection
  - approvals
  - feedback
- AI integrations:
  - enhancement (external models)
  - generation (future)
- Download system (single + bulk)

---

## Core System Capabilities

### RBAC (Access Control)

- Roles: Photographer, Client, Subject
- Context-aware permissions per gallery/session
- Secure access (magic links or authenticated)

---

### Smart Lifecycle Management

- Detects gallery inactivity
- Suggests:
  - archive
  - compression
  - cold storage
  - ZIP export

Goal: reduce storage cost automatically

---

### Quotes & Invoicing (Stripe)

- Pricing attached to:
  - sessions
  - galleries
  - deliverables
- Quote → Invoice → Payment lifecycle
- Integrated into workflow, not separate

---

## What Makes This Different

ProPhoto does not focus on one step.

It removes the friction **between steps**:

- planning → shooting
- shooting → ingest
- ingest → delivery
- delivery → payment

---

## Product Philosophy

- Move decisions upstream (before the shoot)
- Minimize decisions downstream (after the shoot)
- Use context to guide, not replace, the photographer
- Support both:
  - high-automation workflows (calendar-driven)
  - manual workflows (metadata + user control)

---

## Summary

ProPhoto is not a gallery app.

It is a system that understands a shoot and guides the photographer from upload to delivery—without forcing them to think when they’re most likely to make mistakes.
